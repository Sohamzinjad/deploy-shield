DeployShield

A Lightweight Application Deployment Gateway with an Integrated ML-Based Runtime Security Layer

🛡️ Runtime Protection
Intercepts & scores every HTTP request in real-time before reaching app containers.
🤖 Real ML Classifier
Trained models (Random Forest / XGBoost) detecting SQLi, XSS, and Command Injection.
🐳 Single-Host Docker
Deliberately scoped deploy engine prioritizing security engineering depth over cloud fluff.

The Problem

Default application deployments are defenseless against immediate attack traffic.

🔴 Immediate Attack Exposure
Every application deployed online is instantly probed by automated botnets for SQLi, XSS, and command injection patterns.
⚠️ Reliance on Code Perfection
Small teams and student projects deploy without any protective layer, assuming application code is 100% vulnerability-free.
💰 Expensive Enterprise Afterthought
Production platforms bolt on security as a separate, costly enterprise layer (WAFs, CDN filters) rather than integrating it natively into the deploy path.
🎯 The Missing Paradigm
A developer platform where security is a first-class citizen of the deployment pipeline, built-in by default from day one.

The Core Idea

Integrating Machine Learning Threat Classification natively into the deployment path.

💡 Push a Repo ➔ Get a Protected Running App
DeployShield provides a minimal deployment gateway that intercepts every HTTP request, evaluates it against a trained machine learning classifier, and enforces real-time blocking of malicious traffic while forwarding benign traffic transparently.
1. Deploy Engine
Accepts a git URL, builds a local Docker container image, and spawns the application instance automatically.
2. Gateway Proxy
Acts as the single entry reverse proxy intercepting all inbound HTTP methods, URLs, headers, and request bodies.
3. ML Inference Layer
Scores payloads in real-time. Malicious traffic is blocked with HTTP 403 & logged; benign traffic is proxied.

Microservices Breakdown

Modular, decoupled microservice architecture orchestrated via Docker Compose.

🛡️ /gateway (Express Proxy)
Intercepts incoming traffic on /apps/:appId/*, extracts payload metadata, queries ml-service, blocks threats, and proxies benign traffic.
🧠 /ml-service (FastAPI + Py)
Extracts hand-engineered feature vectors (lengths, special char frequency, pattern tokens) and predicts attack probabilities using Random Forest.
⚙️ /api-server (Control Plane)
Central REST API storing container routing tables, deployment states, real-time threat logs, and aggregated security telemetry.
🔨 /build-service (Build Engine)
Clones target repositories, builds Docker images using host socket, runs containers on deployshield-net, and registers container endpoints.

What Makes It a Real Contribution

Focusing engineering depth on ML detection rigor rather than PaaS boilerplate.

1. Real ML Thesis
Trained on labeled HTTP attack datasets (e.g. CSIC 2010), evaluated with Precision, Recall, and F1 per class — not a hardcoded keyword blocklist dressed up as "AI."
2. Own Evaluation Data
Generates custom labeled attack traffic using sqlmap and OWASP ZAP against sandboxed target apps so results prove real-world defense capability.
3. Scoped Platform Engine
Intentionally minimal deployment mechanics (single-host Docker) so engineering time is focused on detection accuracy and false-positive trade-offs.

Evaluation & Measurement Framework

Rigorous metrics defined to benchmark security accuracy & performance.

🎯 Detection Accuracy
Evaluated using Precision, Recall, F1-Score, and Confusion Matrices across specific threat classes (SQLi, XSS, Command Injection).
⚡ False-Positive Rate (FPR)
Measuring FPR on legitimate web traffic — blocking valid users is worse than missing an occasional attack. Target FPR < 0.5%.
⏱️ Latency Overhead
Measuring p50, p95, and p99 millisecond latency added to requests by the ML classification step (target overhead < 15ms).
🔥 Mixed Traffic Stress Testing
Evaluating gateway stability and throughput under concurrent mixed traffic loads (benign browser requests + automated attack scripts).

Scope Boundaries (What This Is Not)

Clear engineering constraints defining the boundaries of our Capstone project.

❌ Not a Commercial PaaS
No multi-region routing, no CDN edge nodes, no custom domain SSL management, and no user billing systems.
❌ Not a Full WAF Replacement
Focuses on a defined set of high-impact attack categories (SQLi, XSS, Cmd Injection), not every layer-7 threat vector.
✅ The Real Value Contribution
Proving the native security integration pattern and evaluating ML classification trade-offs rigorously on custom attack data.
